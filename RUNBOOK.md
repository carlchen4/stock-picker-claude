# RUNBOOK — 月度执行手册

一页纸照着做。**模型和比例都 set-and-forget,每月只"跑 + 对齐名单"。**

## 配置(战略,不每月改)

- **80% CA / 20% US 科技**(加元口径)。
- CA = `picker.py`(5 板块:金融/能源/工业/公用ZUT/材料);US = `picker_us.py`(科技)。
- 比例一年复查一次;**绝不**根据近期表现每月调(=追涨,过拟合)。

---

## 每月(选股日)

```bash
./run.sh monthly                          # 跑两个 picker,出 CA+US 新名单 + 发邮件
./run.sh allocate --ca <加股值> --us <美股值>   # 出买卖清单(80/20 + 股数)
```

1. `monthly` 自动重训(用最新数据)、出名单、写 picks_log、发邮件。**不用调任何参数。**
2. `allocate` 给你:再平衡动作 + 每只该买几股(CA 加元 / US 折美元)。
3. 照清单下单。**执行用市价、当天做完** —— 择时已验证是负期望(`entry_timing.py`),别等、别挂低价。
4. CA 名单等权占 80%,US 名单等权占 20%。

> 当前持仓金额要手输(系统看不到券商,凭证只在本机 = 安全红线)。

---

## 每季度(或偏离 >±5pp 时)

- 再平衡回 80/20。`allocate.py` 会提示是否触发带宽 —— 没超就**不动**。
- 漂移优先用"新钱 + 卖出(如 TSLA)"来纠,少做无谓买卖(省手续费/税)。

## 每年

- 复查 80/20 比例(用 `ca_us_blend.py` 重算)。
- 复查模型是否还成立:看实盘 OOS 记录(`oos_tracker.py`)有没有 edge 衰减,**不是再调回测**。

---

## 什么时候找 Claude(判断,不是算术)

算术交给脚本;**这些才问我:**
- 某只 pick 看着不对劲 / 想了解逻辑
- 大盘有事(FOMC、崩盘)要不要缓一手再平衡
- 卖某只(`./run.sh sell TSLA` 出价格+分析师+事件三段报告)
- 比例/模型要不要做下一轮改进

## 别做的事(已验证为坑)

- ❌ 每月调比例或模型参数
- ❌ 为多收益上 top5 集中 / 银行·黄金 ETF 化 / 加新板块(全 regress,见 PLAN)
- ❌ 把券商凭证放进 repo 或 GitHub Actions
- ❌ 挂限价"抄底"买入(择时负期望)

## 速查

| 命令 | 作用 |
|---|---|
| `./run.sh monthly` | 月度选股(CA+US)+ 邮件 |
| `./run.sh allocate --ca N --us N` | 80/20 配置 + 买卖清单 |
| `./run.sh sell TSLA` | 卖出时机三段报告 |
| `./run.sh backtest` / `rigor` | 回测 / 过拟合审计 |
| `oos_tracker.py` | 实盘 vs 回测追踪 |
